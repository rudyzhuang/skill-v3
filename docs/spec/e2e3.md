# e2e3.md — UI 端到端测试（Skill V3 · ai-e2e3）

| 项 | 约定 |
| --- | --- |
| **skill 安装名** | **`ai-e2e3`** |
| **覆盖阶段** | **`ui_e2e`**（写入 **`.pipeline/stages.json`** 键名 **`ui_e2e`**；对话可用 **ui-e2e**） |
| **规范真源** | 本文 + **`docs/input-spec.md` §8.15** + **`docs/templates/stages.json.template`** |
| **与 smoke 关系** | **HTTP smoke**（**ai-publish-dev3**）保持轻量；**UI 交互验收**由本 skill 承担，**不**合并进 `smoke.cjs` |

---

## 1. 意图

在 **deploy（website/admin）** 或 **build + 安装（mobile android/ios）** 就绪后，按契约 **`test_spec`** 中的 **`ui_scenarios[]`** 执行端到端验收：

| 端 | 执行器 |
| --- | --- |
| **website / admin** | Cursor **Browser MCP**（`cursor-ide-browser`），由外部 Agent 驱动 |
| **mobile**（`sub_platform`: android / ios） | **Dart MCP**（`user-dart`）+ 可选 **`flutter test integration_test`** |

支持 **有限次修复环**（`ui_test_fix`），写回 **`stages.ui_e2e`**，并生成 **`.pipeline/reports/ui-e2e-<session>.md`**。

---

## 2. 阶段链位置

```
… → deploy → smoke → ui_e2e → report
```

- **前置**：`stages.smoke` 为 `completed`+`validation.passed=true`，或 `smoke` 为 `skipped` 且 **`ui_e2e` 配置提供可访问 base_url**。
- **mobile**：另需 `stages.build` 对 `(client_target=mobile, sub_platform)` 产物校验通过，且 **`ui_e2e.mobile.*` 设备/模拟器门闸**通过。

---

## 3. `test_spec` 扩展（契约侧）

在 **`<feature>.test-spec.yaml`**（或 `.yml`）中：

```yaml
required_test_levels:
  - unit
  - integration
  - ui_e2e

ui_scenarios:
  - id: admin-login-001
    client_target: admin
    platform: web          # web | android | ios
    feature_id: feat-001   # 可选；缺省取自契约目录名
    preconditions:
      - deploy.dev.admin_url reachable
    steps:
      - action: navigate
        url: "{base_url}/login"
      - action: fill
        ref: "用户名"
        value: "{test_user}"
      - action: click
        ref: "登录"
    expect:
      - type: url_contains
        value: "/dashboard"
      - type: text_present
        value: "欢迎"
```

**校验（ai-design3 `validate-contract`）**：

- `ui_scenarios` 为数组；每项必填 **`id` / `client_target` / `platform` / `steps`(≥1) / `expect`(≥1)**。
- **`platform`** ∈ `web` | `android` | `ios`。
- **`client_target`** 须为 prd-spec 允许的端；`web` 仅允许 `website` | `admin`；`android`/`ios` 仅允许 `mobile`。
- **`steps[].action`** ∈ `navigate` | `click` | `fill` | `press_key` | `wait` | `snapshot` | `scroll`。
- **`expect[].type`** ∈ `url_contains` | `text_present` | `element_present` | `status_code`（仅 web 且与 navigate 联用）。

## 4. `docs/config.dev.json` · `ui_e2e`

```json
{
  "ui_e2e": {
    "enabled": true,
    "require_smoke_passed": true,
    "web": {
      "website": { "base_url_from": "deploy.services.website.url" },
      "admin": { "base_url_from": "deploy.services.admin.url" }
    },
    "mobile": {
      "sub_platforms": ["android"],
      "android": { "device_id": "", "bundle_id": "" },
      "ios": { "simulator": "", "bundle_id": "" }
    },
    "commands": {
      "ui_test_max_fix_attempts": 3
    },
    "stub_mode": false
  }
}
```

| 键 | 说明 |
| --- | --- |
| **`enabled`** | `false` 时整段 **skipped**（`skip_reason` 必填） |
| **`require_smoke_passed`** | 默认 `true`；`false` 时允许 smoke 跳过但须能解析 base_url |
| **`stub_mode`** | `true` 或 env **`AI_E2E3_SKIP_AGENT=1`**：不调用 Agent/MCP，仅结构校验 + 可选 `flutter test` |
| **`commands.ui_test_max_fix_attempts`** | 默认 **3**，对齐 **ai-code3** `test_max_fix_attempts` |
| **`mobile.android.auto_launch_emulator`** | 默认 **true**；**无已连接设备**（含 USB 真机）时再 `flutter emulators --launch` |
| **`mobile.ios.auto_launch_simulator`** | 默认 **true**（仅 macOS）；同上，优先真机/已启动模拟器 |
| **设备选择顺序** | `device_id` 配置且在线 → **USB 真机** → **已启动模拟器** → 自动 launch AVD/Simulator |
| **不可解阻塞** | `flutter emulators` 列表为空 / Flutter 缺失 / launch 失败 → 退出码 **1**，`blocker=mobile_env_unsatisfied` |
| **`mobile.*.install_before_test`** | 默认 **true**；`flutter build`（若缺产物）→ **`flutter install -d <device>`** |
| **`mobile.commands.boot_wait_s`** | 启动模拟器后等待设备就绪（默认 **90**） |
| **`mobile.commands.smoke_run_s`** | 无 `integration_test/` 时 `flutter run` 冒烟时长（默认 **35**） |

**超时**：`timeouts.stages.ui_e2e_s`（模板默认 **3600**）。

## 5. `stages.ui_e2e` 写回

| 字段 | 说明 |
| --- | --- |
| **`inputs.summary_hash`** | 规范化后的场景列表 + base_url + build 产物指纹 |
| **`inputs.requires_stage`** | 固定 **`smoke`**（逻辑上亦依赖 **build**/**deploy**） |
| **`outputs.scenarios_total` / `passed` / `failed`** | 汇总 |
| **`outputs.results[]`** | `scenario_id`, `platform`, `passed`, `duration_ms`, `error`, `step_failed` |
| **`outputs.report_path`** | `.pipeline/reports/ui-e2e-*.md` |
| **`outputs.fix_attempts`** | 修复环次数 |
| **`outputs.skip_reason`** | enabled=false 或无视图时 |
| **`validation.passed`** | 全部必跑场景通过 |

**退出码**（与 **`input-spec.md` §5** 对齐）：

| 码 | 含义 |
| ---: | --- |
| 0 | 成功或允许的 skip |
| 1 | 前置/配置/门闸 |
| 3 | 超时 |
| 4 | 场景失败（耗尽修复后仍失败） |

stderr：**`failed_step=ui_e2e`**。

## 6. 执行与修复环

1. **`preflight.cjs`**：`stages.json`、config、`ui_e2e` 块、secret-scan（附录 B）、前置阶段。
2. **`ui-e2e.cjs`**：收集契约下全部 **`ui_scenarios`**；解析 **`{base_url}`** 占位符；按平台分组执行。
3. **Web（非 stub）**：`invoke-e2e-agent.cjs` → **`AI_E2E3_AGENT_BIN`**（默认探测 `cursor-agent`），prompt 要求使用 **Browser MCP** 逐步执行并写出 **`AI_E2E3_UI_E2E_OUTPUT`** JSON。
4. **Mobile（脚本层，先于 Agent）**：`mobile-device.cjs` 按 **android/ios** 调用 `flutter devices --machine`；**优先使用已连接真机或已启动模拟器**；仅当无任何设备时再 **`flutter emulators --launch`**；若本机**未安装**对应模拟器镜像则 **退出码 1（不可解）**；随后 build / install / test 或 smoke。
5. **Mobile（非 stub + Agent）**：在设备已启动且 app 已安装前提下，**`invoke-e2e-agent`** 使用 **Dart MCP** 执行 `ui_scenarios`。
6. **Stub / `AI_E2E3_SKIP_AGENT=1`**：仍执行第 4 步设备/安装/测试脚本（不跳过模拟器门闸）。
7. **失败**：`ui-e2e-fix.cjs` 调用 Agent **`ui_test_fix`**（不改契约）；成功后按配置决定是否 **`build`→`deploy`→重跑**（由 **`ai-auto3`** 或人工续跑；本 skill **不**隐式 deploy）。

---

## 7. 与 ai-auto3

- 默认序列在 **`deploy_smoke`** 之后 spawn **`ai-e2e3/scripts/run.cjs --invoked-by-autorun`**。
- **`ui_e2e.enabled === false`** 时跳过 spawn（不失败）。
- **`to-stage=report`** 前须完成或跳过 **`ui_e2e`**。

---

## 8. 禁止项

- **不得**修改 **`docs/contracts/**`** 契约正文（修复环仅改实现源码）。
- **不得**在报告/日志中保存完整鉴权头、密码、完整响应体。
- **不得**将 UI 失败写进 **`stages.smoke`** / **`stages.test`** 的通过字段。
